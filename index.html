核心功能

覆盖网络诊断的每一个维度

带宽测速

多文件大小梯度测试，精准测量上下行带宽。支持 100KB ~ 2MB 多级样本，自动排除缓存干扰。

实时延迟监控

持续 RTT 采样与实时图表，自动计算抖动、最低/最高延迟，可视化网络波动趋势。

DNS 解析测试

对全球主流域名进行 DNS 解析基准测试，评估解析速度与可达性，识别 DNS 劫持与污染。

CPU 性能基准

Worker 侧 CPU 浮点运算基准，反映边缘节点计算能力，辅助评估函数执行效率。

WebSocket 延迟

基于 WebSocket 的全双工 Ping-Pong 测试，精确测量客户端到边缘节点的实时双向通信延迟。

并发压力测试

多级并发请求测试（4/6/8 路并发），验证边缘节点在高负载下的响应能力与排队延迟。

流式吞吐测试

分块传输编码（chunked）吞吐量测试，模拟大文件下载和流媒体场景下的实际传输性能。

丢包率检测

多轮 HEAD 请求采样统计，精准量化网络丢包率，快速定位链路异常与传输质量问题。

全球节点分布

覆盖 Cloudflare 全球 330+ 数据中心，自动选择最优边缘节点进行诊断测试。

安全与协议分析

TLS/ECH/HTTP 协议版本检测、机器人评分、数据中心代理识别等安全面面俱到。

硬件环境探测

客户端屏幕分辨率、CPU 核心数、时区、语言等环境信息一目了然，辅助用户画像分析。

历史统计面板

测速历史记录与 API 端点用量统计，长期追踪网络性能变化趋势，数据持久化存储。

上线前体检

技术架构

轻量、高性能、零运维的边缘计算方案

用户终端 浏览器 HTTP/WebSocket
Cloudflare Workers 全球边缘运行时
Workers KV 边缘键值存储

Worker 运行时

基于 V8 Isolate 的 Serverless 运行时，在 Cloudflare 全球 330+ 数据中心就近执行诊断逻辑。支持 HTTP Fetch、WebSocket 路由和 Streams API。

KV 持久化

Workers KV 提供最终一致的键值存储，用于保存测速历史数据、API 用量计数和用户配置。读取延迟 <10ms（95% 分位），写入异步传播。

WebSocket 通道

利用 Cloudflare Workers 原生 WebSocket 支持，建立客户端到边缘节点的长连接，实现毫秒级双向延迟测量。兼容 RFC 6455。

快速开始

三步部署你的专属网络诊断平台

1

安装 Wrangler CLI 并克隆仓库

确保 Node.js 18+ 环境，安装 Cloudflare 官方命令行工具。

npm install -g wrangler git clone https://github.com/BlueDriftHK/CF-workers-netdiag.git cd CF-workers-netdiag
2

创建 KV Namespace 并绑定

创建用于存储测速历史和用量统计的 KV 命名空间，并记录返回的 namespace ID。

wrangler kv:namespace create "NETDIAG_KV" wrangler kv:namespace create "NETDIAG_KV" --preview

将输出的 namespace ID 填入 wrangler.toml 对应的 kv_namespaces 配置项。

3

部署到 Cloudflare Workers

执行发布命令，Worker 将在全球 330+ 数据中心部署上线。

wrangler deploy

部署完成后访问 https://你的域名.workers.dev 即可使用。

常见问题

你可能关心的一些问题

NetSight Pro 是完全免费的吗？

是的，项目本身完全开源（MIT License），你可以免费部署到自己的 Cloudflare 账户。运行成本取决于你的测速频率和 KV 使用量，Cloudflare Workers 免费计划每天提供 10 万次请求配额，足够个人和中小团队日常使用。

为什么不直接使用 Speedtest 之类的在线测速网站？

传统在线测速网站通常仅提供带宽和基础延迟测试，缺乏 DNS 解析、WebSocket 延迟、CPU 基准、流媒体吞吐、丢包率、协议安全检测等综合诊断能力。NetSight Pro 面向开发者和运维人员，覆盖了网络诊断的全部关键维度，并且数据可持久化，支持长期趋势分析。

部署后如何确保数据安全？

NetSight Pro 运行在你自己的 Cloudflare 账户中，所有测速数据和 API 用量统计都存储在你自己创建的 Workers KV 命名空间下。项目代码完全透明可审计，不会向任何第三方发送数据。建议在 wrangler.toml 中配置合适的 CORS 策略以限制访问来源。

可以同时监控多个站点吗？

可以。你可以在不同域名下独立部署多个 Worker 实例，或者通过 URL 参数指定不同的测试配置。每个 Worker 实例的 KV 命名空间完全隔离，互不干扰。对于企业用户，推荐使用 Cloudflare Workers for Platforms 实现多租户管理。

测试精度受哪些因素影响？

主要受三方面影响：（1）客户端到边缘节点的物理距离和网络路径；（2）测试时段的网络拥塞程度；（3）客户端设备的 CPU 和浏览器性能。NetSight Pro 通过多轮采样取中位数、梯度测试文件等方法尽量消除偶然误差。建议在多个时段重复测试以获得更准确的基线数据。
