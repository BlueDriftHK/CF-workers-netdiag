## 📢 漏洞披露流程

我们遵循负责任的漏洞披露流程：

1. **报告接收**：安全团队收到漏洞报告后，会确认接收并开始评估。

2. **评估与修复**：我们评估漏洞的严重性和影响范围，制定修复方案并开发补丁。

3. **补丁测试**：在私有环境中测试修复方案，确保不引入新问题。

4. **协调披露**：
   - 在修复发布后，我们会公开披露漏洞详情
   - 我们会向报告者致谢（除非报告者要求匿名）
   - 我们会同步更新 [CHANGELOG](./CHANGELOG.md) 和安全公告

5. **用户通知**：通过 GitHub Release Notes 和项目文档通知用户升级。

### 公开披露信息

公开披露将包含：
- 漏洞编号（CVE 如有）
- 漏洞类型和描述
- 影响版本
- 修复版本
- 致谢（如适用）

## 🛡️ 安全措施概述

NetSight Pro 内置了多层安全防护机制：

### 1. 请求层防护

| 措施 | 说明 |
| :--- | :--- |
| **限流保护** | 通用 API 60次/分钟/IP，CPU 测试 3次/分钟/IP |
| **输入校验** | 所有用户输入经过 `parseInt` 钳制，防止注入 |
| **域名白名单** | DNS 代理仅允许 `cloudflare.com`、`google.com`、`github.com` |
| **CORS 配置** | 可通过配置限制允许的来源 |

### 2. 响应层防护

| 措施 | 说明 |
| :--- | :--- |
| **CSP 策略** | 动态 nonce 防止 XSS 攻击 |
| **HSTS** | 强制 HTTPS 连接（`max-age=31536000`） |
| **X-Frame-Options** | 防止点击劫持（`DENY`） |
| **X-Content-Type-Options** | 防止 MIME 类型嗅探（`nosniff`） |
| **Referrer-Policy** | 限制引用来源泄露（`strict-origin-when-cross-origin`） |

### 3. 数据层防护

| 措施 | 说明 |
| :--- | :--- |
| **KV 类型守卫** | `typeof` 检查防止数据投毒 |
| **数据 TTL** | KV 数据 7 天自动过期 |
| **自动清理** | 超出 5 条的旧记录自动删除 |
| **不存储敏感数据** | 仅存储测速结果，不存储用户身份信息 |

### 4. 外部依赖防护

| 措施 | 说明 |
| :--- | :--- |
| **超时控制** | 所有外部 fetch 设置超时（3-5秒） |
| **信号控制** | 使用 `AbortController` 取消超时请求 |
| **错误静默** | 外部 API 失败时返回空值，不影响主服务 |

## 🔐 安全最佳实践

如果您自己部署 NetSight Pro，我们建议：

### 部署安全

1. **使用最新版本**：始终使用最新的稳定版本
2. **配置限流**：根据您的使用场景调整限流阈值
3. **启用 HTTPS**：使用 Cloudflare 提供的 HTTPS（默认启用）
4. **绑定自定义域名**：避免使用公共 `workers.dev` 子域名（可选）

### 环境安全

1. **保护 KV 数据**：KV 中的数据不包含敏感信息，但建议定期清理
2. **监控日志**：定期检查 Worker 日志，发现异常访问模式
3. **限制 CORS**：如非必要，限制 `access-control-allow-origin` 为特定域名

### 代码安全（自行修改时）

1. **输入验证**：所有用户输入必须经过验证和清理
2. **避免 `eval`**：不要使用 `eval()` 或类似动态执行
3. **使用 CSP nonce**：保持动态 CSP nonce 机制
4. **代码审查**：在提交 PR 前进行代码审查

## 🚨 常见安全问题

### 1. 限流绕过
- **风险**：攻击者可能通过大量 IP 或伪造 `cf-connecting-ip` 绕开限流
- **缓解**：建议配合 Cloudflare 的 Bot Management 或 WAF 使用
- **状态**：⚠️ 已知限制，已在文档中说明

### 2. DNS 代理滥用
- **风险**：攻击者可能通过 DNS 代理端点进行 SSRF 攻击
- **缓解**：仅允许白名单域名，且仅支持 HEAD 请求
- **状态**：✅ 已防护

### 3. CPU 资源耗尽
- **风险**：恶意用户可能高频调用 CPU 测试
- **缓解**：独立限流（3次/分钟/IP），限制迭代次数（≤200万）
- **状态**：✅ 已防护

### 4. 信息泄露
- **风险**：响应头可能泄露 Cloudflare 内部信息
- **缓解**：已最小化响应头，仅保留必要信息
- **状态**：✅ 已防护
